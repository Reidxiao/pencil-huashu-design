# 反 AI Slop 检查清单（完整版 · 两层分级）

> 每稿必查。此清单分为两层：**硬性禁止**（99% 是 slop，发现即改）和**需要理由**（结合场景判断，违反时必须自检）。

## 🚨 第一层：硬性禁止（发现即改，无例外）

## ⚠️ 第二层：需要理由（违反时必须自检）

这些模式在特定场景下是合理的，但 Agent 容易因为"默认"或"最熟悉"而选择它们。违反时必须回答自检问题，否则视为 slop。

### 色彩

| ID | 模式 | 何时合理 | 自检问题 |
|----|------|---------|---------|
| CC01 | 高饱和蓝色主调 `#3B82F6` | SaaS、金融、开发者工具 | "我是因为产品调性需要蓝色，还是因为 AI 默认配色？" |
| CC02 | 纯黑/纯白背景 | 极简主义、高端品牌、暗色模式 | "我是因为设计哲学选择纯色，还是因为懒得配色？" |
| CC03 | 大面积灰色背景 | 数据仪表盘、工具类产品 | "灰色是否服务于内容可读性，还是只是'安全'选择？" |

### 排版

| ID | 模式 | 何时合理 | 自检问题 |
|----|------|---------|---------|
| CT01 | Inter/Roboto 做 display | 极简设计、大量文本场景 | "有没有更适合品牌气质的 display 字体？" |
| CT02 | 全部左对齐 | 长文本阅读、文档类 | "标题是否可以居中或右对齐增加节奏？" |

### 布局

| ID | 模式 | 何时合理 | 自检问题 |
|----|------|---------|---------|
| CL01 | 3 列等分布局 | 仪表盘、定价页、功能对比 | "内容是否有更好的排列方式？不等分是否更有节奏？" |
| CL02 | 大量留白 | 极简设计、高端品牌 | "留白是否服务于呼吸感，还是掩盖内容不足？" |
| CL03 | 完美对称 | 正式场合、企业官网 | "是否可以加入一个不对称元素打破规律？" |

### 图形

| ID | 模式 | 何时合理 | 自检问题 |
|----|------|---------|---------|
| CG01 | 3D 按钮 + 阴影 | 游戏化界面、儿童产品 | "是否符合整体设计语言？" |
| CG02 | Stock photo 滥用 | 需要真实感的场景 | "图片是否与内容真正相关？" |
| CG03 | 饼图/折线图花哨 | 数据可视化场景 | "渐变/多色是否帮助理解数据？" |

## ✅ 推荐做法（正面清单）

### 色彩替代方案

| 替代 | 用在哪 | 示例 |
|------|--------|------|
| 单一有品味的颜色 | 主色调 | `#1A1A1A`（近黑）、`#C4A484`（暖棕） |
| 暖灰/冷灰背景 | 背景 | `#F8F7F4`（暖白）、`#F0F4F8`（冷蓝灰） |
| 品牌色做点缀 | accent | 5% 面积使用，不大面积铺开 |
| 真实图片做主视觉 | Hero 背景 | Wikimedia、Met Museum、Unsplash |
| 细微纹理 | 背景 | 噪点纹理 opacity 0.03-0.08 |

### 排版替代方案

| 替代 | 用在哪 | 示例 |
|------|--------|------|
| 衬线 display 字体 | 标题、H1-H3 | Newsreader、Source Serif、Noto Serif、Literata |
| 无衬线 body 字体 | 正文 | Inter（仅 body）、Söhne、Helvetica Neue |
| 字重对比 | 层级 | 标题 700，正文 400，注释 300 |
| 行高舒适 | 正文 | line-height: 1.5-1.7 |

### 布局替代方案

| 替代 | 用在哪 | 示例 |
|------|--------|------|
| 非对称布局 | 内容区 | 左重右轻，或反之 |
| 网格 + 破格 | 打破规律 | 7 列网格中某个元素跨 3 列 |
| 不同尺寸的内容块 | 卡片组 | 有的 1x1，有的 2x1，有的 1x2 |
| 负空间 | 分隔内容 | 不用线，用空白 |

### 图形替代方案

| 替代 | 用在哪 | 示例 |
|------|--------|------|
| Lucide / Material Symbols | 图标 | 线性风格、统一 stroke |
| 真实图片 | 产品/人物 | 不手画，不用 emoji |
| 品牌色应用 | 视觉识别 | 重复 logo 图形、颜色呼应 |

## 🔍 检查流程

### Step 1：远看（5 秒测试）

- 第一眼看到的是什么？
- 整体氛围是"有品味的设计"还是"AI 生成的"？
- 有没有刺眼的紫/蓝/绿？

### Step 2：近看（硬性禁止扫描）

逐项检查第一层硬性禁止清单（C01-C06 / T01-T06 / L01-L04 / G01-G05 / I01-I02）。

### Step 3：自检（需要理由扫描）

检查第二层清单，对每个违反项回答自检问题。回答不了 → 视为 slop → 修改。

### Step 4：对照正面清单

确认使用了推荐做法，而非仅仅避免了禁止项。

## 📋 AI Slop 自检 Prompt

每次完成设计后，用这个 Prompt 自检：

```
请检查当前 .pen 文件中的设计：

第一层（硬性禁止，发现即改）：
1. 紫渐变 / 彩虹渐变背景
2. 霓虹蓝/绿 #00D4FF / #39FF14 主色调
3. emoji 当图标
4. SVG 手画人脸
5. 科技感透视网格背景
6. 圆角卡片 + 左边框 accent 组合
7. 所有元素居中

第二层（需要理由，违反时自检）：
8. 蓝色主色 #3B82F6 — 是产品需要还是 AI 默认？
9. 3 列等分布局 — 是场景需要还是懒得设计？
10. Inter/Roboto 做 display — 是设计选择还是最熟悉？
11. 纯黑/纯白背景 — 是哲学选择还是懒得配色？

如果有违反，用 batch_design 修正。第二层违反需先回答自检问题。
```

## 案例对比

### ❌ AI Slop 示例

```
Hero 区域：
- 深色背景 + 紫色渐变
- "AI-Powered" 标语
- Inter Bold 标题（32px），Inter Regular 正文（30px）
- 3 列等分网格，每个卡片完全相同尺寸

卡片：
- 白色圆角卡片 + 左边 4px #8B5CF6 accent
- 内容：emoji 🔥 + 标题 + 描述

图标：
- 🔥 💡 ✅ ❌ emoji 组合
```

### ✅ 有品味的设计

```
Hero 区域：
- 暖白背景 + 真实产品图
- Newsreader 衬线标题（48px），对齐左
- 非对称布局，左侧文字占 60%，右侧图片占 40%

卡片：
- 部分浅灰背景，部分近白，无左边框
- 颜色变化来自品牌色板

字体：
- Newsreader Display 标题
- Inter Body 正文（16px），注释（14px）

图标：
- Lucide 图标库，stroke-width 统一
```
