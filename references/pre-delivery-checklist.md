# Pre-Delivery Checklist（交付前检查清单）

> 每个设计稿交付前必须逐项检查，全部 ✅ 才能交付。发现 ❌ 项立即修复。

---

## ✅ Pre-Delivery Checklist

### 1. 变量系统

- [ ] **1.1** 所有颜色引用 `$color.*` 变量，无硬编码 hex
- [ ] **1.2** 所有间距引用 `$spacing.*` 变量
- [ ] **1.3** 暗色主题变量已设置（`$color.brand-primary[theme=light/dark]`）
- [ ] **1.4** 字重、圆角等数值也已 token 化

### 2. 品牌元素

- [ ] **2.1** Logo 是真实文件（不是 CSS 剪影 / SVG 手画 / emoji）
- [ ] **2.2** 品牌色应用到位（不是默认蓝 `#1783FF` 或紫 `#8B5CF6`）
- [ ] **2.3** 产品图（如有）是真实图片（不是色块占位）
- [ ] **2.4** 无其他品牌的视觉资产混用

### 3. AI Slop 自检（核心，必须过）

- [ ] **3.1** 无紫渐变 / 彩虹渐变背景
- [ ] **3.2** 无 Inter / Roboto / Open Sans 用于 display 标题
- [ ] **3.3** 无 emoji 当图标 🔥💡✅❌
- [ ] **3.4** 无 SVG 手画人脸
- [ ] **3.5** 无"科技感"透视网格背景
- [ ] **3.6** 无圆角卡片 + 左边框 accent 色组合
- [ ] **3.7** 无 3 列/4 列完全等分布局
- [ ] **3.8** 无所有元素水平垂直居中
- [ ] **3.9** 无高饱和蓝色 `#3B82F6` 大面积铺开
- [ ] **3.10** 无纯黑 `#000` / 纯白 `#FFF` 全屏背景

### 4. 布局与对齐

- [ ] **4.1** `snapshot_layout({ action: "check_alignment", grid: 8 })` 无对齐问题
- [ ] **4.2** `snapshot_layout({ action: "find_overlaps" })` 无重叠元素
- [ ] **4.3** 间距基于 8px 网格（8 / 16 / 24 / 32 / 48 / 64）
- [ ] **4.4** 内边距 / 外边距比例协调
- [ ] **4.5** 移动端适配（如适用）无溢出

### 5. 组件系统

- [ ] **5.1** 复用组件已标记 `reusable: true`
- [ ] **5.2** 变体使用 `ref` + `descendants` 覆盖
- [ ] **5.3** 组件命名符合约定（atomic → molecular → organism）
- [ ] **5.4** 无硬复制组件（应该用 ref instance）

### 6. 交互状态（原型场景）

- [ ] **6.1** 按钮有 hover / active / disabled 状态
- [ ] **6.2** 表单有默认 / 聚焦 / 错误 / 成功状态
- [ ] **6.3** 导航有当前选中状态
- [ ] **6.4** 加载 / 空状态 / 错误状态有设计

### 7. 可访问性

- [ ] **7.1** 颜色对比度符合 WCAG AA（4.5:1 文本 / 3:1 大文本）
- [ ] **7.2** 焦点状态可见（如需要键盘导航）
- [ ] **7.3** 文字大小可读（body ≥ 14px，注释 ≥ 12px）

### 8. 截图确认

- [ ] **8.1** `get_screenshot({ full_page: true })` 视觉效果满意
- [ ] **8.2** 关键页面 / 组件有独立截图存档
- [ ] **8.3** 暗色模式（如有）有独立截图
- [ ] **8.4** 响应式（如有多端）有各端截图

### 9. 文档记录

- [ ] **9.1** 设计决策已记录（存入 review log）
- [ ] **9.2** 已知问题 / limitation 已告知用户
- [ ] **9.3** 5 维度评审报告已生成

### 10. 版本控制（如在 Git 管理下）

- [ ] **10.1** 已 commit（附描述性 commit message）
- [ ] **10.2** commit message 格式：`feat(design): add <section>` / `fix(design): <fix>`
- [ ] **10.3** 工作区干净（无不相关变更）

## ❌ Check Failed 时的处理流程

```
发现问题
  │
  ├─ P0（影响核心功能）
  │    └─ 立即修复 → 重新过 checklist
  │
  ├─ P1（影响体验）
  │    └─ 评估能否快速修复
  │         ├─ 能 → 修复 → 重新过 checklist
  │         └─ 不能 → 告知用户 → 获同意后可交付但标注
  │
  └─ P2（加分项）
       └─ 记录 → 可交付
```
