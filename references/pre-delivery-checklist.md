# Pre-Delivery Checklist（交付前检查清单）

> 每个设计稿交付前必须逐项检查，全部 ✅ 才能交付。发现 ❌ 项立即修复。

---

## ✅ Pre-Delivery Checklist

### 1. 变量系统

- [ ] **1.1** 所有颜色引用 `$color-*` 变量，无硬编码 hex
- [ ] **1.2** 所有间距引用 `$spacing-*` 变量
- [ ] **1.3** 暗色主题变量已设置（如有暗色模式需求）
- [ ] **1.4** 字重、圆角等数值也已 token 化

### 2. 品牌元素

- [ ] **2.1** Logo 是真实文件（不是 CSS 剪影 / SVG 手画 / emoji）
- [ ] **2.2** 品牌色应用到位（不是默认蓝 `#1783FF` 或紫 `#8B5CF6`）
- [ ] **2.3** 产品图（如有）是真实图片（不是色块占位）
- [ ] **2.4** 无其他品牌的视觉资产混用

### 3. AI Slop 自检

**第一层（硬性禁止，发现即改）**：
- [ ] **3.1** 无紫渐变 / 彩虹渐变背景
- [ ] **3.2** 无 emoji 当图标
- [ ] **3.3** 无 SVG 手画人脸
- [ ] **3.4** 无"科技感"透视网格背景
- [ ] **3.5** 无圆角卡片 + 左边框 accent 色组合
- [ ] **3.6** 无所有元素水平垂直居中

**第二层（需要理由，违反时已自检）**：
- [ ] **3.7** 如使用蓝色主色 → 已确认是产品需要，非 AI 默认
- [ ] **3.8** 如使用 3 列等分布局 → 已确认是场景需要，非懒设计
- [ ] **3.9** 如使用 Inter/Roboto 做 display → 已确认是设计选择
- [ ] **3.10** 如使用纯黑/纯白背景 → 已确认是哲学选择

### 4. 布局与对齐

- [ ] **4.1** `snapshot_layout({ problemsOnly: true })` 无重叠/裁剪问题
- [ ] **4.2** 间距基于 8px 网格（8 / 16 / 24 / 32 / 48 / 64）
- [ ] **4.3** 内边距 / 外边距比例协调
- [ ] **4.4** 移动端适配（如适用）无溢出

### 5. 组件系统

- [ ] **5.1** 复用组件已标记 `reusable: true`
- [ ] **5.2** 实例用 `C()` 创建，`descendants` 定制
- [ ] **5.3** 组件 ID 语义化（`{功能}-{层级}`，如 `hero-title`、`feature-card`）
- [ ] **5.4** 无 `card1`、`card2`、`card-copy-3` 等无意义 ID
- [ ] **5.5** 无硬复制组件（应该用 `C()` 创建实例）

### 6. 命名规范

- [ ] **6.1** Frame ID 语义化：`header`、`hero-section`、`features-grid`
- [ ] **6.2** 组件 ID 语义化：`button-primary`、`card-article`、`input-text`
- [ ] **6.3** 文本 ID 语义化：`hero-title`、`card-description`、`footer-copyright`
- [ ] **6.4** 无数字后缀命名（`card1`→`card-article`，`text2`→`hero-subtitle`）

### 7. 交互状态（原型场景）

- [ ] **7.1** 按钮有 hover / active / disabled 状态
- [ ] **7.2** 表单有默认 / 聚焦 / 错误 / 成功状态
- [ ] **7.3** 导航有当前选中状态
- [ ] **7.4** 加载 / 空状态 / 错误状态有设计

### 8. 可访问性

- [ ] **8.1** 颜色对比度符合 WCAG AA（4.5:1 文本 / 3:1 大文本）
- [ ] **8.2** 焦点状态可见（如需要键盘导航）
- [ ] **8.3** 文字大小可读（body ≥ 14px，注释 ≥ 12px）

### 9. 截图确认

- [ ] **9.1** `get_screenshot` 视觉效果满意
- [ ] **9.2** 关键页面 / 组件有独立截图存档
- [ ] **9.3** 暗色模式（如有）有独立截图
- [ ] **9.4** 响应式（如有）有各端截图

### 10. 文档记录

- [ ] **10.1** 设计决策已记录
- [ ] **10.2** 已知问题 / limitation 已告知用户
- [ ] **10.3** 5 维度评审报告已生成

## ❌ Check Failed 时的处理流程

```
发现问题
  │
  ├─ P0（影响核心功能）
  │    └─ 立即修复 → 重新过 checklist
  │
  ├─ P1（影响体验）
  │    └─ 评估能否快速修复
  │         ├─ 能 → 修复 → 重新过 checklist
  │         └─ 不能 → 告知用户 → 获同意后可交付但标注
  │
  └─ P2（加分项）
       └─ 记录 → 可交付
```
